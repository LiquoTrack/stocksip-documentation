### 4.1.3. Software Architecture ###

#### 4.1.3.1. Software Architecture Context Level Diagrams ####

El diagrama de contexto en la arquitectura de software proporciona una visión general de alto nivel del sistema dentro de su entorno, mostrando cómo interactúa con los actores externos (usuarios, sistemas externos, etc.).

Para StockSip, el diagrama de contexto incluye los siguientes actores y sistemas externos:

<img src="https://i.imgur.com/4RGrUxA.png" alt="context-diagram" style="width: 100%; height: auto;"/>

Para nuestro sistema StockSip, los actores y sistemas externos son:

- **Actores**:
  - **Liquor Store Owner**: El propietario de la tienda de licores que utiliza StockSip para gestionar su inventario y ventas.
  - **Liquor Suppliers**: Proveedores de licores que suministran sus productos y venden a través de la plataforma.
  - **Visitors**: Visitantes que interactuan con la Landing Page para obtener información sobre el sistema.


- **Aplicación**:
  - **StockSip**: La aplicación principal que gestiona el inventario, las ventas y las relaciones con los proveedores.


- **Sistemas Externos**:
  - **Google Services**: Servicios de Google utilizados para diversas funcionalidades, como autenticación y envío de correos.
  - **Cloudinary**: Servicio de gestión y almacenamiento de imágenes.
  - **Mercado Pago**: Plataforma de pagos utilizada para procesar transacciones.
