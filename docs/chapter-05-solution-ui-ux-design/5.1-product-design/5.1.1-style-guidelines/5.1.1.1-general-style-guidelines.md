### 3.1.1. Style Guidelines ###

#### 3.1.1.1. General Style Guidelines ####

La propuesta visual de **StockSip** para dispositivos móviles se basa en transmitir profesionalismo, accesibilidad y tecnología confiable para negocios que requieren control eficiente de su inventario, especialmente licorerías y proveedores. Las decisiones tomadas en cuanto a branding, tipografía, colores y espaciado buscan crear una experiencia clara, funcional y moderna, optimizada para pantallas táctiles y uso en movimiento.

**Branding**

- "StockSip" se presenta como una marca moderna y profesional, enfocada en la gestión inteligente de licores desde dispositivos móviles.

- El logotipo de StockSip presenta un diseño minimalista con formas geométricas que combinan elementos relacionados con la gestión de inventario (como formas angulares que evocan organización y simulan una copa de licor) con una identidad visual limpia y contemporánea.

- El uso de la paleta de colores basada en tonos vibrantes (rojos intensos, rosas suaves y neutros elegantes) proyecta modernidad, dinamismo y eficiencia, manteniendo la profesionalidad necesaria para el entorno empresarial.

- La identidad visual está optimizada para ser reconocible en iconos de aplicaciones móviles y pantallas de diferentes densidades de píxeles.

**Tipografía**

-   **Títulos**: “Poppins” — serif elegante, ideal para transmitir distinción y modernidad.
    
-   **Cuerpo de texto**: “Inter” — sans-serif moderna, clara y altamente legible.
    
-   **Botones y microtextos**: “Roboto Medium” — neutral y funcional, facilita la interacción en interfaces digitales.
    
-   **Elementos destacados** usan una variante "bold" o "semi-bold", asegurando énfasis sin sobrecargar visualmente.

- Los tonos oscuros crean una sensación de exclusividad, mientras que los acentos cálidos (como el rojo quemado) dan vitalidad y dinamismo.

- La armonía entre el color vino y tonos neutros facilita un diseño elegante sin perder claridad.

**Espaciado**

- **Margen mínimo**: 8dp - Para elementos muy cercanos

- **Espaciado estándar**: 16dp - Entre elementos relacionados

- **Espaciado de sección**: 24dp - Entre grupos de contenido

- **Margen de pantalla**: 16-20dp - Márgenes laterales para contenido principal

- **Espaciado generoso**: 32-48dp - Para separar secciones principales

- El diseño sigue Material Design Guidelines para garantizar consistencia en el ecosistema Android.
<br>

**Tono de la comunicación**
    
- **Serio pero accesible**: transmitimos confianza y precisión sin ser excesivamente técnicos.
    
- **Formal/Casual equilibrado**: nos dirigimos a usuarios emprendedores o gerentes de negocios, con un tono que mezcla profesionalismo y cercanía.
    
- **Respetuoso**: mantenemos un lenguaje claro y respetuoso, enfocado en resolver problemas del usuario.
    
- **Sereno con un toque entusiasta**: buscamos reflejar eficiencia con mensajes motivadores que inviten a la acción sin ser invasivos.
<br>

**Botones y elementos táctiles**

- **Altura mínima**: 48dp - Cumple con las recomendaciones de accesibilidad táctil

- **Bordes redondeados**: 8dp - Para botones principales

- **Área táctil**: Mínimo 48x48dp - Garantiza interacción cómoda

- **Estados visuales claros**: Normal, pressed, disabled, focused

- **Elevación**: 2-4dp para botones flotantes y elementos elevados

**Patrón de diseño móvil:** 

- Se utiliza el patrón de navegación **Side Navigation (Drawer)** ubicado en el lateral derecho para las funciones principales de la aplicación.
-
- **Tab Navigation** para secciones secundarias dentro de cada módulo principal.
-
- **Cards** como contenedores principales para mostrar información de inventario.
-
- **Pull-to-refresh** y **Infinite scroll** para listas de productos.
<br>


**Responsive Design**

La interfaz se adapta automáticamente a diferentes densidades y tamaños de pantalla:

- **mdpi**: ~160 dpi (densidad base de referencia)

- **hdpi**: ~240 dpi (1.5x)

- **xhdpi**: ~320 dpi (2x)

- **xxhdpi**: ~480 dpi (3x)

- **xxxhdpi**: ~640 dpi (4x)
<br>

**Componentes y patrones compatibles**

- **Side Navigation Drawer**: Navegación principal deslizable desde el lateral derecho con menú completo

- **Cards**: Contenedores con elevación de 2-4dp, radio de esquina 8dp

- **FAB (Floating Action Button)**: Para acciones principales (agregar producto)

- **Swipe Actions**: Deslizar para acciones rápidas (editar, eliminar)

- **Pull-to-Refresh**: Para actualizar listas de inventario

- **Snackbar**: Para feedback temporal y acciones de deshacer
<br>

**Navegación Móvil**

- **Side Navigation Drawer (Izquierda)**: Panel deslizable desde el borde derecho para secciones principales (Inventario, Alertas, Perfil, Configuración)

- **Tabs**: Para subsecciones dentro de cada módulo

- **App Bar**: Con título de sección actual y botón para abrir drawer

- **Back Navigation**: Consistente con patrones del sistema operativo


**Estados de Carga y Feedback**

- **Skeleton Screens**: Para tiempos de carga de contenido

- **Progress Indicators**: Circulares para operaciones indefinidas, lineales para procesos con progreso

- **Empty States**: Ilustraciones y mensajes motivacionales cuando no hay contenido

- **Error States**: Mensajes claros con opciones de recuperación



**Accesibilidad Móvil**

Todos los elementos siguen las pautas de accesibilidad móvil:

- **Tamaño mínimo táctil**: 48dp x 48dp

- **Contraste**: Mínimo 4.5:1 para texto normal, 3:1 para texto grande

- **Soporte para lectores de pantalla**: Content descriptions apropiadas

- **Navegación por teclado**: Para usuarios con dispositivos de entrada alternativos

- **Zoom**: Soporte hasta 200% sin pérdida de funcionalidad
